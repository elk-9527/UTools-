## 一、这是什么

一款 **uTools 插件**：在 Windows 桌面上显示一个**毛玻璃悬浮窗**，内含 5 个可自由拖拽/缩放的小组件（时钟、日历、天气、待办、快捷启动）。

- **所属仓库**：`C:\Users\Lenovo\Desktop\UTools`（git 根），内含两款插件
  - `cc-skip/`：第一款，Claude Code 快速启动器，已发布 v1.0.0
  - `Win11 桌面组件/desktop-widget/`：**本项目**，第二款，开发中
- **技术栈**：纯原生 HTML/CSS/JS（无框架、无构建），uTools API + Electron preload
- **代码规模**：约 2555 行（9 个 .js + 6 个 .css + 2 个 .html）

## 三、技术架构（一句话理解）

```
index.html (插件入口，常驻进程)
   └─ utools.createBrowserWindow → 创建独立悬浮窗 widget.html
                                      └─ framework.js  注册表 + 拖拽/缩放/吸附 + 布局持久化
                                           └─ 5 个 module（继承 module-base.js 的 WidgetModule）
preload.js  → 给前端暴露 Node 能力（启动 exe / 开文件夹 / 开网址 / 取屏幕尺寸）
```

- **窗口**：无边框、透明、置顶（`screen-saver` 层级）、跳过任务栏
- **数据**：全部存 `utools.db`（支持跨设备云同步）
  - `widget-layout`：各模块位置/尺寸
  - `weather-city`：天气城市经纬度
  - `todo-*`（_id 前缀）：待办项，`type: 'todo'`，含 `date` 字段
  - `ql-*`（_id 前缀）：快捷方式，`type: 'shortcut'`
- **视觉**：CSS `backdrop-filter: blur+saturate` 毛玻璃 + 多层渐变高光模拟液态玻璃，深浅色随系统 `prefers-color-scheme` 自动切换
- **模块通信**：用 `window` 上的 CustomEvent，目前有 `calendar:dateSelect`（日历→待办）

### 🔴 #1 + #2 日历 ↔ 待办联动补全

这两项联动紧密，建议一起做。

**A. 日历显示"哪天有待办"的小圆点**

`modules/calendar/calendar.js`：

1) 新增方法，读取所有有待办的日期：
```js
_getTodoDates() {
  try {
    const docs = utools.db.allDocs('todo-');      // 前缀查询，见 #6
    const set = new Set();
    for (const d of docs) if (d.date) set.add(d.date);
    return set;
  } catch (e) { return new Set(); }
}
```

2) 在 `_renderCalendar()` 渲染"当月日期"的循环里，给有待办的日子加 `has-todo` 类（并补 `data-date`，顺便给下面筛选用）：
```js
const todoDates = this._getTodoDates();          // 循环前取一次
// ... 当月日期循环内：
const dateStr = `${this._currentYear}-${String(this._currentMonth + 1).padStart(2, '0')}-${String(d).padStart(2, '0')}`;
let cls = 'calendar-day';
if (/* 是今天 */) cls += ' today';
if (todoDates.has(dateStr)) cls += ' has-todo';
html += `<div class="${cls}" data-day="${d}" data-date="${dateStr}">${d}</div>`;
```

3) 待办变化时让日历重绘圆点——`render()` 里监听，`destroy()` 里解绑：
```js
render(container) {
  super.render(container);
  this._renderCalendar();
  this._todoChangedHandler = () => this._renderCalendar();
  window.addEventListener('todo:changed', this._todoChangedHandler);
}
destroy() {
  if (this._todoChangedHandler) window.removeEventListener('todo:changed', this._todoChangedHandler);
  super.destroy();
}
```

`modules/todo/todo.js`：在 `_addTodo()`、`_deleteTodo()`、`_toggleTodo()` 末尾各派发一次事件：
```js
window.dispatchEvent(new CustomEvent('todo:changed'));
```

### 🟡 #4 模块级配置（以时钟为例打样）

基类已留 `onConfigChange(key, value)`，落地一个范例即可推广：

`modules/clock/clock.js`：
```js
constructor() { super({...}); this._showSeconds = true; }
render(container) {
  super.render(container);
  const cfg = utools.db.get('cfg-clock'); if (cfg) this._showSeconds = cfg.showSeconds !== false;
  // …渲染时按 this._showSeconds 决定是否拼接 :ss
}
onConfigChange(key, value) {
  if (key === 'showSeconds') {
    this._showSeconds = value;
    const ex = utools.db.get('cfg-clock'); const doc = { _id: 'cfg-clock', showSeconds: value };
    if (ex && ex._rev) doc._rev = ex._rev; utools.db.put(doc);
    this._update();
  }
}
```
触发入口可放在卡片右键菜单或 #3 的设置面板里调用 `module.onConfigChange('showSeconds', false)`。

## 八、给接手者的注意事项

- **改样式前先确认类名**：本项目 js 用模板字符串拼 HTML，类名散落在 js 里，改 css 时务必和对应 js 的 `className` / `innerHTML` 对齐（已核实当前 js↔css 类名一致，不要被旧印象误导）。
- **新增模块四步**：① 建 `modules/<name>/` 写 js+css ② 继承 `WidgetModule` ③ 在 `widget.html` 引入并 `WidgetFramework.register(new XxxModule())` ④ 若要默认显示，加进 `framework.js` 的 `_presetModules`。
- **数据读写统一走 `utools.db`**：`put` 前要带上已有文档的 `_rev` 否则冲突（现有模块都处理了，照抄即可）。
- **调试**：右键悬浮窗 → 开发者工具；无构建步骤，改完直接重载。
